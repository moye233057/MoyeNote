# 一、异常概念。
```
1、异常是指程序运行过程中出现的非正常现象，例如用户输入错误、除数为零、要处理的文件不存在、数组下标越界的情况。
2、异常是用于解决逻辑代码和错误代码放一起，导致程序员需要考虑例外事件的情况。
3、异常处理指：程序在出现问题时依旧能够正确执行剩余的程序，而不会因为异常终止程序执行。
4、运行程序出现错误在结果窗口会出现:
Traceback: 追溯
most recent call back:  最后一次调用
具体的错误位置及错误原因
```
# 二、常见异常类别
```
pyhthon引入了很多描述和处理异常的类，称为异常类
异常	描述
NameError	尝试访问一个没有申明的变量
ZeroDivisionError	除数为0
SyntaxError	语法错误
IndexError	索引超出序列范围
KeyError	请求一个不存在的字典关键字
IOError	输入输出错误（比如你要读的文件不存在）
AttributeError	尝试访问未知的对象属性
ValueError  数值错误
TypeError 类型错误
IndexError 索引越界异常
```
# 三、异常捕获：
```
try:
   pass
except NameError:
   pass
except SyntaxError:
   pass
else:  # 没有异常执行else
   pass
finally:  # 不管有没有异常都执行
   pass

assert 断言
assert是一句等价于布尔真的判定，发生异常就意味着表达式为假。
什么是使用断言的最佳时机？
防御性的编程
运行时对程序逻辑的检测

程序中的常量
检查文档
```

# 四、with上下文管理
```
with上下文管理可以自动管理资源
在with代码执行完毕后自动还原进入该代码之前的现场或上下文。不论何种原因跳出with语句，不论是否发生异常，总能保证资源的正确释放，极大简化工作。
应用场景: 文件操作、网络通信相关场合。
```
